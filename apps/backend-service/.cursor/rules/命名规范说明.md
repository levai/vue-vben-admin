# Model 命名规范

## 📋 命名规则

### Entity（实体类）

- **规则**: `Sys` + 业务名称（首字母大写）
- **示例**: `SysDept`, `SysMenu`, `SysRole`, `SysUser`
- **说明**: 实体类对应数据库表，使用 `Sys` 前缀表示系统表

### DTO（数据传输对象）

- **规则**: 业务名称（首字母大写） + `DTO`
- **示例**: `DeptDTO`, `MenuDTO`, `RoleDTO`, `LoginDTO`
- **说明**: DTO 用于接收前端请求数据，不包含数据库相关字段（如 `id`, `createTime`, `deleted`）

### VO（视图对象）

- **规则**: 业务名称（首字母大写） + `VO`
- **示例**: `DeptVO`, `MenuVO`, `RoleVO`, `UserVO`, `LoginResultVO`
- **说明**: VO 用于返回给前端的数据，包含需要展示的字段

## 📝 命名原则

1. **一致性**: 同一类型的对象使用相同的命名规则 ✅
2. **可读性**: 名称应该清晰表达对象的用途 ✅
3. **简洁性**: 避免不必要的中间词（如 `Info`） ✅
4. **明确性**: 如果简化会导致混淆，可以保留更明确的名称 ✅

## ⚠️ 注意事项

1. **新增 Model 类时**，请遵循上述命名规范
2. **Entity 类**必须使用 `Sys` 前缀，对应数据库表名
3. **DTO 类**用于接收前端请求，不包含数据库字段
4. **VO 类**用于返回给前端，统一使用 `VO` 后缀
5. **避免使用中间词**（如 `Info`, `Detail`），保持命名简洁
